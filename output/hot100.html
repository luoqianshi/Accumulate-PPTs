Positioning

这份手册到底在解决什么问题

A

适用对象

目标用户是准备互联网研发岗笔试的研究生，默认已有编程基础，但还没有建立题型与套路之间的映射关系。

03
Stage 1

第一阶段：哈希、位运算、链表基础

3 个示例题

哈希表

两数之和、字母异位词分组、最长连续序列。关键词是 空间换时间，把查找另一半降到 O(1)。

3 个示例题

位运算

只出现一次的数字、多数元素、颜色分类。目标不是炫技，而是形成 lowbit、异或抵消、三指针分区的肌肉记忆。

9 个示例题

链表基础

反转、判环、合并、两数相加、删除倒数第 N 个节点等，核心是 哑节点 + 快慢指针 + 三指针翻转。

这一阶段的目标不是“刷很多”，而是把最常见的线性结构操作压成模板。尤其链表题，一旦你熟悉 dummy、slow/fast、pre/cur/nxt，很多题的恐惧感会立刻下降。

  • 代表简单题：#1 两数之和、#136 只出现一次的数字、#206 反转链表
  • 代表中等题：#2 两数相加、#142 环形链表 II、#19 删除倒数第 N 个节点
  • 手册建议用时：3-5 天，适合作为第一轮启动器。
04
Stage 2

第二阶段：双指针、滑动窗口、栈

4 个示例题

双指针

移动零、盛最多水的容器、三数之和、接雨水。关键是判断谁该移动，以及移动后不丢失最优解。

5 个示例题

滑动窗口

无重复子串、异位词、和为 K 的子数组、最小覆盖子串、滑动窗口最大值。核心是 扩右、缩左、维护状态。

5 个示例题

栈

有效括号、最小栈、每日温度、最大矩形、字符串解码。重点是理解单调栈在“找最近更大/更小元素”上的统一解释。

容易卡住

随机刷的典型问题

  • 不知道 left 什么时候移动，right 什么时候停止。
  • 窗口里该记录频次、位置还是答案，逻辑混乱。
  • 单调栈只会背题，不会抽象成“维护有序边界”。
正确策略

按模板分组学习

  • 双指针先分同向 / 相向，窗口先分定长 / 不定长。
  • 把“窗口状态”写成可维护变量，再决定缩放条件。
  • 把单调栈当作“延迟结算机制”，不是死记题名。
05
Stage 3

第三阶段：树形结构与搜索能力补齐

13 个示例题

二叉树

前中后序、层序、BST 验证、最近公共祖先、最大路径和。重点是递归定义与返回值设计。

5 个示例题

二分查找

标准二分、边界二分、旋转数组、二维矩阵，把“有序区间折半缩小”练熟。

3 个示例题

堆

第 K 大、前 K 高频、中位数数据流。核心是维护 top-k 与双堆平衡。

9 个示例题

数组 / 矩阵

Kadane、区间合并、乘积数组、缺失正数、螺旋矩阵、旋转图像等，训练原地操作与结构映射。

这一阶段信息量最大，建议按“遍历模板 → 返回值含义 → 代表难题”来拆。比如树题，前序适合构造，中序适合 BST，后序适合汇总信息；只要把遍历与任务绑定，树题就会变得清晰很多。

06
Stage 4

第四阶段：回溯、贪心、图论 / BFS / DFS

8 个示例题

回溯

全排列、子集、组合总和、括号生成、单词搜索、分割回文串、N 皇后。公式就是 做选择 → 递归 → 撤销选择。

4 个示例题

贪心

股票、跳跃游戏、划分字母区间。核心不是“局部最优”这句口号，而是证明当前决策不会伤害后续最优。

4 个示例题

图论 / BFS / DFS

岛屿数量、腐烂的橘子、课程表、Trie。把图看成“节点 + 邻接关系”，再决定用 BFS 还是 DFS。

如果说前几阶段是在练“如何操作结构”，这一阶段开始训练的是“如何搜索答案空间”。一旦会抽树、图、状态图，很多看上去完全不同的题会突然归并成同一类。

07
Stage 5

第五阶段：动态规划是一种状态设计能力

10 个示例题

一维 DP

爬楼梯、打家劫舍、零钱兑换、单词拆分、LIS、最长有效括号等，先掌握一条线上的状态转移。

5 个示例题

多维 DP

不同路径、最小路径和、最长回文子串、最长公共子序列、编辑距离，把二维表格理解成状态空间。

5 个示例题

高级链表

K 个一组翻转、随机链表复制、排序链表、合并 K 链表、LRU 缓存，属于链表综合题与数据结构设计题。

2 个综合技巧

其他高频技巧

下一个排列、寻找重复数。这类题往往考察你对“结构重解释”的熟练程度。

DP 的心法在手册里写得很清楚：定义状态 → 推导转移 → 确定初始值 → 确定遍历顺序。很多人不是不会写代码，而是第一步状态就定义错了。

  • 建议先吃透 #198 打家劫舍、#322 零钱兑换、#300 LIS 这三道一维代表题。
  • 二维 DP 先理解“表格每个格子代表什么”，再谈转移方程。
  • 手册给出的第五阶段预计用时是 7-10 天，也是全流程最需要耐心的一段。
08
Pattern Matrix

最后真正要记住的是这十类高频套路

哈希表

O(1) 查找

典型触发词是“找另一半”“判重”“频次统计”。代表题：两数之和、异位词、最长连续序列。

双指针

线性逼近

适合有序数组、容器面积、区间碰撞、链表中点。关键是决定谁移动、为什么移动。

滑动窗口

维护窗口

适合子串、区间、定长 / 不定长连续问题。模板核心是 right 扩展、left 收缩、状态更新。

单调栈

最近更大/更小

每日温度、柱状图最大矩形、滑窗最大值，本质上是延迟结算与边界确定。

二分查找

有序空间

不是只会找值，还要会找左边界、右边界，以及“满足条件的最小 / 最大位置”。

BFS / DFS

图与树搜索

BFS 适合最短步数、分层扩散；DFS 适合连通块、递归遍历、搜索整棵答案树。

回溯

答案树

排列、组合、子集、棋盘搜索，全靠“路径、选择列表、结束条件、撤销选择”四件套。

贪心

局部最优

难点是证明。遇到区间、覆盖、跳跃、买卖时，要先判断是否存在单调的最优决策。

动态规划

状态转移

能否拆成子问题？子问题是否重复？状态和决策是否有限？这是 DP 的三连问。

并查集 / 结构重构

少量高频变体

手册附录中把并查集列为变体套路；另外像 Floyd 判圈、原地置换，也是常见的结构重解释题型。

09
Templates

模板代码不是抄答案，而是压缩思维开销

Hash / Prefix
seen = {}
for i, x in enumerate(nums):
    if target - x in seen:
        return [seen[target - x], i]
    seen[x] = i
一眼识别“找另一半”时直接套，核心是把 O(n) 查找换成哈希命中。
Sliding Window
left = 0
for right, ch in enumerate(s):
    update_in(ch)
    while not valid():
        update_out(s[left])
        left += 1
    refresh_answer()
窗口题最重要的不是代码量，而是明确 valid() 到底代表什么条件。
Backtracking
def dfs(path, choices):
    if finish(path):
        ans.append(path[:])
        return
    for c in choices:
        choose(c)
        dfs(path, choices)
        undo(c)
组合、排列、子集都从这棵答案树出发，再补去重、剪枝和 start 索引。
Dynamic Programming
dp[0] = base
for i in range(1, n + 1):
    dp[i] = best(
        transition_1,
        transition_2,
        ...
    )
先定义 dp[i] 的语义，再写转移；不要先看代码再倒推状态。
10
Sprint Strategy

怎么把这本手册真正变成笔试得分

错误节奏

看似努力，实际低效

  • 今天刷树，明天刷 DP，后天随机开一题，知识点无法沉淀。
  • 只看题解不手写模板，遇到相似题仍然没有起手式。
  • 把时间花在少数困难题上，却没有补齐中频套路的覆盖率。
推荐节奏

首轮学习 + 高频复盘

  • 第一轮按 5 阶段顺序完整推进，建立稳定题型地图。
  • 每天先背 2-3 个模板，再做对应代表题，形成输入输出闭环。
  • 考前只复盘高频模板、典型边界和困难题心法，不重新开荒。

如果完全按手册建议时长推进，首轮大约需要 25-36 天；如果时间更紧，就优先保证阶段 1 到阶段 3 的模板熟练度，因为那部分覆盖了最多的中低风险得分题。

11
Checklist

考前最后复盘，只看这四件事

01

能否秒判题型

看到题目后，能不能在 20 秒内判断它更像哈希、双指针、窗口、栈、树搜索还是 DP。

02

是否有起手模板

哈希、窗口、DFS/回溯、二分、堆、DP 这些骨架，最好都能脱稿写出第一版。

03

边界是否敏感

空数组、单节点、重复元素、越界、初始值、更新顺序，这些往往比算法本身更容易丢分。

04

有没有一句话解释

每题都能讲清“为什么这么做”，说明你已经从背答案升级为掌握套路。

真正的目标不是把 Hot 100 每一题都记成独立案例，而是把它们压缩成一组可以迁移、可以组合、可以复用的算法语言。
