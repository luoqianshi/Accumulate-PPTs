这是一份面向互联网研发岗笔试准备的算法学习汇报稿。原始材料把 Hot 100 拆成 5 个阶段、18 个知识模块与一套可复用模板；这份 HTML PPT 的目标，是把长文内容压缩成一套能快速讲清学习路径、重点套路与冲刺节奏的展示版本。

01
Overview

这份手册在解决什么问题

A

目标用户

面向准备互联网研发岗笔试的计算机专业研究生。默认读者有编程基础，但对刷题体系还缺少整体地图，常见问题是题会看、手却起不来。

B

使用方式

强调按顺序学习，而不是随机刷题。先用 3-5 个典型题形成套路感，再反复背模板代码，让常见题型从“认识”升级为“可快速输出”。

C

核心原则

原文给出的主张非常明确：先掌握套路，再理解原理，最后追求优雅。也就是说，第一优先级是能在笔试场景里先写出来。

这不是一本追求“每题最优理论证明”的教材，而是一份把高频算法题工程化、模板化、可复盘化的通关脚本。

03
Stage 1

第一阶段：哈希、位运算、链表基础

3 个示例题

哈希表

核心作用是把查找从 O(n) 压到 O(1)。代表题包括两数之和、字母异位词分组、最长连续序列，强调“空间换时间”的直觉。

3 个示例题

位运算技巧

异或消去、lowbit、右移等都是常见基础招式。重点不是炫技，而是识别哪些题适合用位运算做常数级简化。

9 个示例题

链表基础

反转链表、判环、合并、删除倒数节点、回文链表等题高度依赖哑节点、快慢指针和三指针翻转模板。

这一阶段建议用时为 3-5 天。真正要带走的不是题量，而是三个基础反射：查找先想哈希、链表先想 dummy、结构判断先想快慢指针。

  • 代表简单题：#1 两数之和、#136 只出现一次的数字、#206 反转链表
  • 代表中等题：#2 两数相加、#142 环形链表 II、#19 删除链表倒数第 N 个节点
  • 阶段关键词：O(1) 查找、异或抵消、哑节点、快慢指针、三指针翻转
04
Stage 2

第二阶段：双指针、滑动窗口、栈

4 个示例题

双指针

移动零、盛最多水的容器、三数之和、接雨水。难点在于决定谁移动，以及为什么移动不会错过更优解。

5 个示例题

滑动窗口

无重复字符最长子串、最小覆盖子串、滑动窗口最大值等题，核心是先定义窗口状态，再决定扩张与收缩条件。

5 个示例题

栈

有效括号、每日温度、柱状图最大矩形、字符串解码等题，统一视角是利用栈维护局部有序边界与待结算信息。

常见误区

随机刷题时容易卡住

  • 不知道 left 与 right 何时移动，边界逻辑频繁出错。
  • 窗口里到底要记录频次、位置还是答案，思路不稳定。
  • 单调栈只会背题名，不会抽象为“最近更大 / 更小”的通用模型。
推荐方法

按模板分组记忆

  • 双指针先区分同向 / 相向，窗口先区分定长 / 不定长。
  • 先写“窗口状态变量”，再写 while 条件和更新答案的位置。
  • 把单调栈理解成一种延迟结算机制，而不是零散技巧合集。
05
Stage 3

第三阶段：树形结构、二分、堆与结构搜索

13 个示例题

二叉树

最大深度、层序遍历、BST 验证、最近公共祖先、最大路径和等，关键是把遍历顺序和返回值语义绑定起来。

5 个示例题

二分查找

从标准二分扩展到左右边界、旋转数组和二维矩阵，训练的是“在有序空间里折半定位”的稳定感。

3 个示例题

堆

第 K 大、前 K 高频、数据流中位数，背后其实都是 top-k 维护和双堆平衡问题。

9 个示例题

数组 / 矩阵

Kadane、合并区间、乘积数组、缺失正数、螺旋矩阵、旋转图像，考的是原地操作和结构映射。

这一阶段信息量最大，也是最容易“看懂但写不稳”的阶段。建议按“遍历模板 → 返回值含义 → 代表难题”来拆解：例如树题里，前序更适合构造，中序更适合 BST，后序更适合汇总信息。

06
Stage 4

第四阶段：回溯、贪心、图论 / BFS / DFS

8 个示例题

回溯

全排列、子集、组合总和、括号生成、单词搜索、N 皇后等题都可归入“答案树搜索”，核心公式是做选择、递归、撤销选择。

4 个示例题

贪心

股票、跳跃游戏、划分字母区间。看似简单，真正难点在于证明当前局部最优不会伤害未来全局最优。

4 个示例题

图论 / BFS / DFS

岛屿数量、腐烂的橘子、课程表、Trie 等题，训练的是如何把问题重新抽象成节点与关系，再决定遍历方式。

前几阶段更多是在练“怎么操作结构”，这一阶段开始练“怎么搜索空间”。一旦会把问题画成树、图或状态图，很多看似不同的题会自然归类到同一种方法。

07
Stage 5

第五阶段：动态规划本质是状态设计能力

10 个示例题

一维 DP

爬楼梯、打家劫舍、零钱兑换、单词拆分、LIS、最长有效括号等，核心是先把一条线上的状态转移推熟。

5 个示例题

多维 DP

不同路径、最小路径和、最长回文子串、最长公共子序列、编辑距离，把二维表格当作状态空间看待。

5 个示例题

高级链表

K 个一组翻转、随机链表复制、排序链表、合并 K 链表、LRU 缓存，属于综合结构题与设计题。

2 个补充技巧

其他高频技巧

如下一个排列、寻找重复数，虽然数量少，但常在笔试中承担“区分熟练度”的作用。

原文把 DP 心法总结为四步：定义状态 → 推导转移 → 确定初始值 → 确定遍历顺序。很多题不是不会写，而是第一步状态定义就偏了。

  • 建议先吃透 #198 打家劫舍、#322 零钱兑换、#300 最长递增子序列
  • 二维 DP 必须先说清每个格子代表什么，再谈如何从左上、上方或左方转移过来。
  • 这一阶段建议用时 7-10 天，是整个学习流程里最考验耐心的一段。
08
Pattern Matrix

附录真正沉淀下来的，是十类高频解题套路

哈希表

O(1) 查找

适合找另一半、判重、频率统计与分组归类，典型题是两数之和、字母异位词、最长连续序列。

双指针

线性逼近

适合有序数组、容器面积、区间碰撞、链表中点等问题，关键是判断谁动、为何动。

滑动窗口

维护区间

本质是连续区间维护，所有窗口题都绕不开“扩右、缩左、更新状态”三步。

单调栈

边界结算

每日温度、柱状图最大矩形、窗口最大值，都可以理解为寻找最近更大或更小元素。

二分查找

有序空间

除了找值，还要会找边界、插入位置，以及满足条件的最小或最大位置。

BFS / DFS

遍历搜索

BFS 擅长最短步数与层序扩散，DFS 擅长递归遍历、连通块统计与答案树搜索。

回溯

搜索答案树

排列、组合、子集、棋盘问题都适合用路径、选择列表、结束条件、撤销选择来统一理解。

贪心

局部最优

并非所有题都能贪心，关键是能否证明当前最优选择对后续仍然安全。

动态规划

状态转移

是否存在重叠子问题、可定义有限状态、可写出递推关系，是判断 DP 的核心问题。

并查集 / 结构重构

少量变体

附录中也提醒了并查集与结构重解释类题的价值，如 Floyd 判圈、原地置换等思路。

09
Templates

模板代码的作用，是降低思维开销与起手难度

Hash Table
seen = {}
for i, num in enumerate(nums):
    if target - num in seen:
        return [seen[target - num], i]
    seen[num] = i
适用于“两数之和”类题目。关键不是代码本身，而是看到“找另一半”时能瞬间想到用哈希把查找降到 O(1)。
Sliding Window
left = 0
for right, ch in enumerate(s):
    update_in(ch)
    while not valid():
        update_out(s[left])
        left += 1
    refresh_answer()
窗口题最容易乱的不是写法，而是 valid() 的定义。只要状态定义对，模板本身非常稳定。
Backtracking
def dfs(path, choices):
    if finish(path):
        ans.append(path[:])
        return
    for c in choices:
        choose(c)
        dfs(path, choices)
        undo(c)
排列、组合、子集、括号生成、本质上都在同一棵答案树里走，只是剪枝条件和去重方式不同。
Dynamic Programming
dp[0] = base
for i in range(1, n + 1):
    dp[i] = best(
        transition_1,
        transition_2,
        ...
    )
DP 模板真正要背的是思考顺序：先定义 dp[i] 的语义，再写转移，而不是看完代码后倒推状态。
10
Sprint Strategy

如何把这本手册真正转化为笔试得分

低效节奏

看似努力，实际分散

  • 今天刷树，明天刷 DP，后天又随机开一题，知识点无法沉淀成系统。
  • 只看题解不手写模板，遇到相似题时还是没有稳定的起手动作。
  • 花太多时间啃少数困难题，却忽略了中频套路的覆盖率与熟练度。
推荐节奏

首轮顺刷 + 高频复盘

  • 第一轮严格按 5 个阶段顺序推进，先建立完整题型地图。
  • 每天背 2-3 个模板，再做对应代表题，把输入与输出绑在一起。
  • 考前只复盘高频模板、边界细节与困难题心法，不再大面积随机开荒。

若完全按原文建议推进，首轮学习约需 25-36 天。若时间压缩，则优先保证阶段 1 到阶段 3 的熟练度，因为这部分覆盖了最多、最稳定的笔试得分点。

11
Checklist

考前复盘，只检查这四件事

01

能否快速判题型

看到题目后，能否在 20 秒内判断它更像哈希、双指针、窗口、栈、树搜索还是 DP，这决定了起手速度。

02

是否有起手模板

哈希、窗口、DFS / 回溯、二分、堆、DP 这些骨架是否能脱稿写出，是判断复习质量最直观的标准。

03

边界是否敏感

空数组、单节点、重复元素、初始值、越界、更新顺序，这些细节往往比算法本身更容易导致丢分。

04

能否一句话讲清思路

如果每题都能用一句话解释“为什么这么做”，说明你已经从背答案升级为掌握方法。

真正的目标，不是把 Hot 100 记成 100 个孤立题，而是把它们压缩成一套可以迁移、可以组合、可以复用的算法语言。
