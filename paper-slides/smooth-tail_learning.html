ICCV 2023 NUS & HIT

Boosting Long-tailed Object Detection
via Step-wise Learning on Smooth-tail Data

基于平滑尾部数据逐步学习的长尾目标检测增强方法

核心贡献：构建平滑尾部数据（Smooth-tail Data），设计逐步学习框架统一微调与知识迁移，在 LVIS 数据集上显著提升了稀有类别的检测精度。

Na Dong, Yongqiang Zhang, Mingli Ding, Gim Hee Lee

ABSTRACT

论文摘要

Real-world data tends to follow a long-tailed distribution, where the class imbalance results in dominance of the head classes during training.
研究动机

现实数据往往呈现长尾分布，类别不平衡导致训练过程中头部类别主导模型，尾部类别性能严重退化。

方法概述
  • 构建 Smooth-tail Data：将长尾分布平滑衰减，缓解对头部类别的偏置
  • 在完整长尾数据上预训练，保留所有类别的判别能力
  • 在 Head-dominant 数据上微调，获得头部类别专家模型
  • 在 Tail-dominant 数据上训练统一模型，同时迁移头部专家知识
核心结果：LVIS v0.5 上 ResNet-50 AP 从 27.0% 提升至 30.3%，稀有类别 AP 从 15.5% 提升至 24.9%。

研究背景与动机

长尾分布问题
  • 现实数据集类别数量呈极端不平衡
  • 头部类样本充足，尾部类样本极少
  • 模型训练被头部类别主导
  • 尾部类别检测精度显著下降
现有方法分类
  • 数据重采样：RFS、Bi-level sampling
  • 损失重加权：EQLv2、Seesaw Loss、EFL
  • 增量学习：LST（Hu et al.）
  • 度量学习：AHRL
LST 的核心思想
  • "Divide & Conquer" 策略
  • 将长尾数据按样本量排序并硬分割
  • 分 N 个阶段进行类增量少样本学习
  • 每阶段合并上一阶段的exemplar replay
LST 采用多阶段增量学习，但存在灾难性遗忘、决策边界不一致等问题。

LST 方法的三大缺陷

1. 硬分割导致判别性降低
  • 外观相似的类别被分到不同阶段
  • 仅在 exemplar replay 子集上共同训练
  • 类别间判别能力显著下降
2. 决策边界不一致
  • 当前模型同时训练头部和尾部的 replay
  • 与仅训练头部的前一模型边界差异大
  • 阻碍头部知识保持与尾部学习
3. 多阶段加速灾难性遗忘
  • 数据集被切分为众多小平衡子集
  • 知识迁移步骤增多
  • 灾难性遗忘被显著加速
图1：本文方法（上）vs LST（下）。本文仅需3步，LST需N个阶段。
关键区别：本文方法始终维持所有类别的数据参与训练，仅调整各类别的占比来构建 Smooth-tail 数据。

Smooth-tail 数据构建

类别划分
  • 给定长尾数据集 Dl，含 Cl 个类别
  • Head 类别 Chead：每类包含 ≥ M 张图像（M=30）
  • Tail 类别 Ctail：每类包含 < M 张图像
Dhead：Head-dominant 数据
  • Head 类别的平衡子集（主导）
  • 少量 Tail 类别的平衡子集（辅助）
  • 缓解 Head 类内部不平衡
  • 确保与最终模型决策边界差距小
Dtail：Tail-dominant 数据
  • Tail 类别的平衡子集（主导）
  • 少量 Head 类别的平衡子集（replay）
  • 防止统一模型遗忘 Head 知识
图3：Smooth-tail 数据构建示意图
置信度引导 Exemplar 选择

使用预训练模型预测的分类分数排序，选择 top-scoring 实例作为代表性 exemplar。同时优先选择分数 > 0.5 且来自不同图像的实例，保证多样性。

Step 0：预训练
  • 在完整 Dl 上预训练
  • 使用 Deformable DETR 损失
  • 学习所有类别的判别表示
  • 为后续步骤提供分类置信度
Step 1：Dhead 微调
  • 仅更新 class-specific 模块
  • Projection layer Φp + Classification head Φcls
  • 保持 class-agnostic 模块冻结
  • 获得 Head class expert
Step 2：Dtail 训练
  • 统一模型初始化自 Head expert
  • 仅更新 class-specific 模块
  • Feature-level 知识蒸馏
  • Classification-level 知识蒸馏
设计原则：通过限制可更新参数（仅 class-specific 模块），防止特征表示严重偏移，同时利用知识蒸馏保持头部类别性能。

预训练与 Dhead 微调

预训练（Step 0）
  • 基于 Deformable DETR 框架
  • 在完整长尾数据 Dl 上训练 50 epochs
  • 初始学习率 2×10-4，40 epoch 衰减 0.1
  • 提供分类置信度用于 exemplar 选择
  • 保留所有类别的判别分离能力
Dhead 微调（Step 1）
  • 仅更新 Φp（投影层）和 Φcls（分类头）
  • Class-agnostic 模块（特征提取器、Transformer）完全冻结
  • 使用标准 Hungarian Loss 微调 1 epoch
  • 学习率 2×10-5
Hungarian Loss
Lhg = ∑i [ Lcls(ci, p̂σ(i)) + 1{ci≠∅} Lbox(bi, b̂σ(i)) ]

Lcls：Sigmoid Focal Loss
Lbox：L1 + GIoU Loss
σ(i)：最优二分匹配索引

关键约束：冻结 class-agnostic 模块对先前表示施加强约束，确保判别性表示在后续过程中不会严重偏移。
Dhead 数据配比

Head 类：每类 200 个实例（主导）
Tail 类：每类 30 个实例（辅助）
采用 Step-wise RFS 缓解子集内不平衡

知识转移：Dtail 训练

图5：知识转移框架示意图
训练设置
  • 统一模型初始化自 Head expert
  • 仅更新 Φp 和 Φcls，其余冻结
  • 训练 2 epochs，lr = 2×10-4
  • 1 epoch 时 lr 衰减 0.1
Feature-level 蒸馏

使用 Head class focused binary mask：

Lfm_dis = (1/2Nhead) ∑i,j,k maskheadij || funifyijk - fheadijk ||2

mask 在 Head 类 GT 框内为 1，外部为 0
防止对 Tail 类学习产生负面影响

Classification-level 蒸馏

共享 Object Query 特征对齐预测：

Lcls_dis = KL_div( log(p̂unifyshared), p̂head )

qhead：Head expert 的 decoder 输出
直接共享给统一模型，避免 query 不匹配

总损失函数
Ltotal = Lhg + λfmLfm_dis + λclsLcls_dis

λfm = 0.1，λcls = 1

实验设置

数据集
  • LVIS v0.5：1230 类，57K 训练图，5K 验证图
  • LVIS v1.0：1203 类，100K 训练图，19.8K 验证图
  • 类别划分：Frequent (>100)、Common (10-100)、Rare (<10)
  • 评估指标：APb、APr、APc、APf
基线模型
  • 基于 Deformable DETR
  • Backbone：ResNet-50 / ResNet-101
  • ImageNet 预训练权重初始化
训练配置
  • 8 × RTX 3090 GPUs，每卡 batch size = 2
  • 优化器：AdamW，weight decay = 10-4
  • Step 0：50 epochs，lr = 2×10-4
  • Step 1：1 epoch，lr = 2×10-5
  • Step 2：2 epochs，lr = 2×10-4
  • 阈值 M = 30，λfm = 0.1，λcls = 1

核心结论
  • Smooth-tail 数据有效缓解了极端类别不平衡
  • 逐步学习框架统一了微调与知识迁移
  • 在 LVIS v0.5 和 v1.0 上均达到 SOTA
  • 稀有类别检测精度提升最为显著
局限性与观察
  • 频繁类别 APf 略有下降（31.6 → 30.9）
  • 可能因 Dtail 中频繁类代表性不足
  • Dhead 中平衡 head 类也可能导致频繁类欠表示
该方法简单而有效，通过预训练保留全类别判别力，再分步微调与知识迁移，实现了对长尾数据的优雅处理。

总结与展望

本文提出了一种基于平滑尾部数据的逐步学习框架，通过预训练、微调和知识迁移三阶段策略，在长尾目标检测任务上取得了显著的性能提升，尤其在稀有类别上表现突出。
